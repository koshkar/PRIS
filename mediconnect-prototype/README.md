# MediConnect — прототип

Прототип универсальной платформы обмена медицинскими данными между
пациентами, врачами и администраторами. Сделан для демонстрации сценариев
из ранее построенных диаграмм компонентов / activity / sequence
(см. `Preview.wsd` в корне репозитория).

## Что реализовано

- Три роли: **Пациент / Врач / Администратор** (раздельные интерфейсы).
- **Auth-сервис**: вход по логину/паролю, сравнение роли с заявленной
  при логине (защита от эскалации роли).
- **Medical Records Service**: чтение/создание записей (анализы, приёмы,
  назначения), фильтрация по пациенту.
- **Consent Service**: управление согласиями пациент → врач, отзыв
  согласия в один клик.
- **Передача данных врачу** с обязательной проверкой согласия — без
  активного согласия передача блокируется и фиксируется в журнале аудита.
- **Notification Service**: уведомления врачу о новых данных,
  автоматическая отметка «просмотрено».
- **Audit Service**: журнал событий безопасности (входы, отказы,
  выдача/отзыв согласия, передача, создание записей) с фильтром.
- Состояние хранится в `localStorage` — это эмулирует и серверную БД,
  и оффлайн-кэш мобильного клиента.

## Запуск

Локально:

```sh
cd mediconnect-prototype
python3 -m http.server 8080
# открыть http://localhost:8080
```

Развёртывание: статический сайт, разворачивается на любой бесплатный
хостинг — GitHub Pages, Netlify, Vercel, Cloudflare Pages.
